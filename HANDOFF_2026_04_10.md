## 2. MCP 코드 인덱서

### 무엇을 했나
- `mcp/` 디렉토리에 MCP 서버 구현 (Node.js + @modelcontextprotocol/sdk)
- game_core.js를 인덱싱하여 필요한 함수/섹션만 추출 → 토큰 절약

### 제공하는 도구
| 도구 | 용도 |
|------|------|
| `get_overview` | 프로젝트 구조 + 섹션맵 + 함수 목록 |
| `get_function` | 함수명으로 코드 추출 |
| `get_section` | 섹션별 코드 추출 |
| `search_code` | 키워드/정규식 검색 |
| `get_char_data` | CSV 캐릭터/마법카드 조회 |
| `get_recent_changes` | git log 요약 |
| `get_lines` | 특정 줄 범위 추출 |

### 새 컴퓨터에서 설정
1. Node.js 필요 (v22+ 권장)
2. `cd mcp && npm install`
3. `.claude/settings.json`의 MCP 서버 경로에서 node.exe 경로를 해당 컴퓨터에 맞게 수정

### settings.json 현재 내용
```json
{
  "mcpServers": {
    "babg-index": {
      "command": "C:\\Users\\prees\\node-v22.15.0-win-x64\\node.exe",
      "args": ["mcp/index.js"],
      "cwd": "C:\\claude\\babg"
    }
  }
}
```
→ 다른 컴퓨터에서는 `command`를 해당 환경의 node 경로로 변경 필요

## 3. records.json 인코딩 복구

### 무엇이 있었나
- Worker 초기 버전에서 `atob()`이 UTF-8 한글을 깨뜨림
- 게임이 깨진 Worker를 통해 records.json을 4회 저장 → 데이터 누적 손상
- `82260fa` 커밋(마지막 정상 버전)으로 복구
- Worker를 GitHub API raw media type(`application/vnd.github.v3.raw`)으로 변경하여 해결

### 잃어버린 데이터
- `82260fa` 이후 ~ 복구 시점 사이에 저장된 전적 기록 (약 4건)은 깨져서 복구 불가
